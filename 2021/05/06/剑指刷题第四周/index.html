剑指刷题第四周

Posted on 2021-05-06 In Algorithm

2021.4.26~2021.5.2

剑指33 - 二叉搜索树的后序遍历序列

二叉搜索树：若左子树不为空，则左子树上所有节点均小于根节点，若右子树不为空，则右子树上所有结点均大于根节点

二叉搜索树的中序遍历：直接数组排序

思路一：先求中序遍历，然后重建二叉树(剑指07)

思路二：直接从后序遍历构建二叉树

剑指34 - 二叉树中和为某一个值的路径

思路：直接遍历二叉树的每一个节点，将合法节点加入路径

ps：从本题中应学会怎么dfs遍历一棵二叉树

剑指35 - 复杂链表的复刻

思路：在原链表每一节点都插入一个新节点，则random的复刻就是random->next，再恢复原链表即可

剑指36 - 二叉搜索树与双向链表

思路：根节点拎出来，递归左右子树，左右子树分别返回首位节点，将三部分拼接起来

  • 当前节点是叶节点，直接返回当前节点
  • 左右子树都有，先递归处理左子树，再递归处理右子树，拼接
  • 左子树存在，右子树不存在
  • 右子树存在，左子树不存在

剑指37 - 序列化二叉树

思路：可以采用不同的遍历方式，这里采用前序遍历的顺序

剑指38 - 排列数字

思路：暴力搜索问题

暴力搜索的顺序是什么，按照什么样的顺序去暴搜比较好

假设有两个1，那么规定第二个1必须放到第一个1后面，来避免重复的情况

剑指39 - 数组中出现次数超过一半的数字

思路：消消乐的思路，设置val和cnt两个变量，从前往后遍历，要是遍历的值和val相同，cnt++，不相同则cnt–，要是cnt<0，接下来遍历到的值设为val

剑指40 - 最小的k个数

思路：维护一个大根堆，存的是最小的k个数，堆顶元素存的是当前k个数的最大值，遍历数组，要是遍历到的数比堆顶小，弹出堆顶，放入该数，时间复杂度为O(nlogk)

剑指41 - 数据流的中位数

思路：维护两个集合，一个小根堆和一个大根堆，小根堆存放集合中较大的那一半数，大根堆存放集合中较小的那一半，两个堆的堆顶就是集合的中位数，每次插入都先默认插入大根堆(很重要的一个思想，先直接插入，然后再调整，在这里有一半的概率是不用调整的，就算要调整，也不亏，节省了时间），要是大根堆堆顶比小根堆堆顶大，交换两个堆顶，要是大根堆的size比小根堆的size大2，弹出大根堆堆顶放入小根堆

  • 逆序：交换
  • 下多：转移

剑指42 - 连续子数组的最大和

思路一：动态规划

  • s，存的是以前一个数为结尾的子数组的最大和

    • s > 0, s + x
    • s <= 0, x

    思路二：分治

剑指43 - 从1到n整数中1出现的次数

思路：从高位到低位，计算每一位的1出现的次数，复杂度O(logn)
